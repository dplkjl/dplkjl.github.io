Looping!

Why Loop?

A loop is a fundamental way to repeat the execution of certain pieces of code in your programme. You wouldn't want to have to write the same piece of code over and over just to collect multiple bits of information from the user, or to display something that has lots of values. Instead you write one piece of code and loop through it a certain number of times.

What are the choices?

There are four loop constructs in c#. They are the 'while', 'do', 'for' and 'foreach' loop constructs:

while Loops

This loop construct continues executing an embedded statement as long as a certain condition holds true. It's syntax is:

while (condition) embedded-statement

The while statement evaluates it's condition before entering the loop body on every iteration.

do...while Loops

The 'do' loop is similar to the above 'while' loop except that the expression is evaluated after the body of the loop has completed. By implication this means that it will always run at least once. It's syntax is:

do embedded-statement
 while (condition);

for Loops

The 'for' loop executes the body of code a certain number of times. It has three parts, an initialiser. a condition and an iterator. It's syntax is:

for (initialiser; condition; iterator) embedded-statement

foreach Loops

The 'foreach' loop executes the body of code for all elements found in a collection of some sort. It's syntax is:

foreach (type identifier in expression) embedded-statement

(Note that the curly brackets are optional surrounding the embedded-statements in all of these looping structures.)

How do I choose?

Loops allow you to execute a block of statements repeatedly. C# offers several statements to construct loops with, including the while, do, for, and foreach loops. while loops execute a block of statements as long as an expression is true, do loops execute a block of statements at least once and then keep going as long as a condition is true, for loops execute a block of statements a specified amount of times, and foreach loops execute a block of statements for each item in a collection. Normally a block of statements will execute from beginning to end. However, the normal flow of a loop can be changed with the break and continue statements.
